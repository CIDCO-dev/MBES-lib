Revision history

Rev I.01 09 MAY 2022

  • Quality factor calculation
    - Added section about total uncertaintiy and quality factor.

  • Supported sensor input
    - Updated sound velocity sensor input with examples.

  • #MRZ datagram EMdgmMRZ_def
    Description changed:
    - EMdgmMRZ_def.SIsample_desidB - Typo fixed. The BS corrections applied at the centre sample should be reflectivity1_dB, not reflectivity2_dB.

  • #SVT datagram EMdgmSVT_def
    Modified fields:
    - EMdgmSVTinfo_def.sensorInputFormat - Support for Valeport NMEA and XDR sound velocity added.

  • #SCL datagram EMdgmSCL_def
    Modified fields:
    - EMdgmSCLdataFromSensor_def.clockDevPU_microsec - Field changed from nanoseconds to microseconds due to problems with overflow.

Rev I. 04 JUN 2021

  • Document information
    - All former revisions have been added.
    - Revision history cleaned up. Links added to related fields.

  • Multibeam data logging
    - Description addded of how logged multibeam datagrams are partitioned and how to combine them to a complete multibeam datagrams.

  • Common #Sxx (sensor) datagrams
    Description changed:
    - EMdgmScommon_def.sensorStatus - Description changed to clarify data OK and sensor active.

  • #SKM datagram EMdgmSKM_def
    Description changed:
    - EMdgmSKMinfo_def.sensorInputFormat - Code Octupus MCOM removed from the list due to it has never been supported.
    Modified fields:
    - EMdgmSKMinfo_def.sensorDataContents - Bit number 3 is now only heave. Was previously heave and vertical velocity. Bit number for error field (bit number 5) replaced by delayed heave.
    - KMbinary_def.status - The status of error fields (bit number 5, 21) are deprecated. Changed to delayed heave as the official KM Binary sensor input format. Bit number 6, 22 for delayed heave will still be present.
    - EMdgmSKMinfo_def.sensorStatus - Description changed to clarify data OK and sensor active. Bit number 1 is used for marking the sensor is chosen as active attitude velocity if using attitude split (one sensor input as attitude only, and one sensor input for attitude velocity).

  • #SVT datagram EMdgmSVT_def
    Description changed:
    - EMdgmSVTinfo_def.sensorStatus - Description changed to clarify data OK and sensor active.

  • #SPO datagram EMdgmSPOdataBlock_def
    Description changed:
    - EMdgmSPOdataBlock_def.correctedLat_deg - Corrected position is now added for inactive sensors.
    - EMdgmSPOdataBlock_def.correctedLong_deg - Corrected position is now added for inactive sensors.
    - EMdgmSPOdataBlock_def.speedOverGround_mPerSec - Speed is now added for inactive sensors.
    - EMdgmSPOdataBlock_def.courseOverGround_deg - Course over ground is now added for inactive sensors.
    - EMdgmSPOdataBlock_def.ellipsoidHeightReRefPoint_m - Ellipsoid height is now added for inactive sensors.

  • #CPO datagram EMdgmCPOdataBlock_def
    Description changed:
    - EMdgmCPOdataBlock_def.correctedLat_deg - Corrected position is now added for inactive sensors.
    - EMdgmCPOdataBlock_def.correctedLong_deg - Corrected position is now added for inactive sensors.
    - EMdgmCPOdataBlock_def.speedOverGround_mPerSec - Speed is now added for inactive sensors.
    - EMdgmCPOdataBlock_def.courseOverGround_deg - Course over ground is now added for inactive sensors.
    - EMdgmCPOdataBlock_def.ellipsoidHeightReRefPoint_m - Ellipsoid height is now added for inactive sensors.

  • #SDE datagram EMdgmSDE_def, updated to version 1
    New fields:
    - EMdgmSDEdataFromSensor_def.depthRaw_m - Raw depth reading from sensor

  • #MRZ datagram EMdgmMRZ_def, updated to version 3
    New functionality:
    - To support dual head, dual swath, EMdgmMbody_def is added to every multibeam datagram partition for logging purposes. Note, these fields are removed in the final *.kmall file. See Multibeam data logging for details.

  • #MWC datagram EMdgmMWC_def, updated to version 2
    New functionality:
    - To support dual head, dual swath, EMdgmMbody_def is added to every multibeam datagram partition for logging purposes. Note, these fields are removed in the final *.kmall file. See Multibeam data logging for details.

Rev H. 03 APR 2020

  • #MRZ datagram EMdgmMRZ_def, updated to version 2
    New fields:
    - EMdgmMRZ_pingInfo_def.activeModes - New variable. Showing active modes. Currently used for EM MultiFrequency Mode only. Replaced EMdgmMRZ_pingInfo_def.padding4
    Modified fields:
    - EMdgmMRZ_pingInfo_def.frequencyMode_Hz - Added new frequencies for EM 2040 and EM 2040PM.

Rev G. 01 NOV 2019

  • #FCF datagram EMdgmFCF_def, new datagram for backscatter calibration

  • #MRZ datagram EMdgmMRZ_def, updated to version 1
    New fields:
    - EMdgmMRZ_pingInfo_def.bsCorrectionOffset_dB - Backscatter offset set in the installation menu.
    - EMdgmMRZ_pingInfo_def.lambertsLawApplied - Beam intensity data corrected as seabed image data (Lambert and normal incidence corrections).
    - EMdgmMRZ_pingInfo_def.iceWindow - Ice window installed.
    - EMdgmMRZ_pingInfo_def.padding4 - Padding for byte alignment.
    - EMdgmMRZ_txSectorInfo_def.highVoltageLevel_dB - 20log(Measured high voltage power level at TX pulse or Nominal high voltage power level). This parameter will also include the effect of user selected transmit power reduction (transmitPower_dB) and mammal protection. Actual SL = txNominalSourceLevel_dB + highVoltageLevel_dB. Unit dB.
    - EMdgmMRZ_txSectorInfo_def.sectorTrackingCorr_dB - Backscatter correction added in sector tracking mode. Unit dB.
    - EMdgmMRZ_txSectorInfo_def.effectiveSignalLength_sec - Signal length used for backscatter footprint calculation. This compensates for the TX pulse tapering and the RX filter bandwidths. Unit second.
    Description changed:
    EMdgmMRZ_txSectorInfo_def.txNominalSourceLevel_dB - Actual SL = txNominalSourceLevel_dB + highVoltageLevel_dB. Unit dB re 1 microPascal.

  • #MWC datagram EMdgmMWC_def, updated to version 1
    New fields:
    - EMdgmMWCrxBeamData_def.detectedRangeInSamplesHighResolution - The same information as in EMdgmMWCrxBeamData_def.detectedRangeInSamples with higher resolution. Two way range in samples. Approximation to calculated distance from tx to bottom detection [meters] = soundVelocity_mPerSec * detectedRangeInSamples / (sampleFreq_Hz * 2). The detected range is set to zero when the beam has no bottom detection.

  • #SVT datagram EMdgmSVT_def
    Modified fields:
    - EMdgmSVTinfo_def.sensorInputFormat - Kongsberg sound velocity datagrams KSSIS 80 and KSSIS 43 added.
